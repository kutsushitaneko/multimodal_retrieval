# Ver2.1.0 ファイル名連動機能 - 仕様書と作業計画

## 概要
「検索と回答生成」タブのギャラリーで選択されている画像のファイル名を、「イメージ管理」タブの「検索結果からコピー」ボタンをクリックすることで、「イメージ管理」タブのファイル名エリアに連動させる機能を実装する。

## 機能仕様

### 1. 新規UIコンポーネント
- **ボタン名**: 「検索結果からコピー」
- **配置場所**: イメージ管理タブのファイル名入力欄の近く
- **ボタンスタイル**: 通常のボタン（variant="secondary"）
- **初期状態**: 無効（interactive=False）

### 2. 機能要件

#### 2.1 ボタンの有効/無効制御
- **有効になる条件**: 検索タブで画像が選択されている場合
- **無効になる条件**: 
  - 検索タブで画像が選択されていない場合
  - アプリケーション起動直後

#### 2.2 ファイル名コピー機能
- **動作**: ボタンクリック時に検索タブで選択された画像のファイル名をイメージ管理タブのファイル名入力欄に設定
- **データソース**: `state['selected_result']['file_name']`
- **対象コンポーネント**: `filename_input`

#### 2.3 エラーハンドリング
- 画像が選択されていない場合: 適切なメッセージ表示
- データが不正な場合: エラーメッセージ表示

### 3. ユーザビリティ要件
- ボタンクリック後、ファイル名欄にフォーカスが移る
- コピー成功時には視覚的フィードバックを提供
- 既存のワークフローを阻害しない

```python
# app/ui/events.py 内
def copy_filename_from_search_result(self, state_data):
    """検索結果から選択された画像のファイル名をコピーする"""
    if state_data is None or 'selected_result' not in state_data:
        return gr.Textbox(interactive=True), "❌ 検索結果で画像を選択してください。"
    
    selected_result = state_data['selected_result']
    if 'file_name' not in selected_result:
        return gr.Textbox(interactive=True), "❌ ファイル名情報が取得できません。"
    
    filename = selected_result['file_name']
    return gr.Textbox(value=filename, interactive=True), f"✅ ファイル名「{filename}」をコピーしました。"
```

### 3. ボタン状態制御
```python
def update_copy_button_state(self, state_data):
    """検索結果の選択状態に応じてコピーボタンの状態を更新"""
    has_selection = (state_data is not None and 
                    'selected_result' in state_data and 
                    'file_name' in state_data['selected_result'])
    
    return gr.Button("検索結果からコピー", variant="secondary", interactive=has_selection)
```

### 問題: 回答生成ボタンで質問文がVLMに渡されない
**発生日時**: 2024年7月5日
**症状**: 回答生成ボタンを押した際に質問文入力エリアの内容がVLMに正しく渡されていない

### 調査結果
1. **Event Listenersの確認**
   - `generate_answer`関数のパラメータは正しく設定されている
   - `answer_question_input`が第1パラメータとして正しく渡されている

2. **検索時の質問文自動設定の確認**
   - 検索ボタンのイベントハンドラーに質問文更新処理が実装されている（lines 204-208）
   - 質問文の更新処理が別のイベントとして登録されている

3. **問題の原因**
   - 検索処理と質問文更新処理のタイミングの問題
   - 検索処理完了後に質問文が更新される可能性

### 修正計画
1. **検索ボタンイベントの修正**
   - 検索処理と質問文更新を同一チェーン内で実行
   - 質問文更新のタイミングを検索処理開始時に変更

2. **イベント登録順序の最適化**
   - 質問文更新を検索処理の最初に配置
   - 検索結果が表示される前に質問文が設定されるよう調整

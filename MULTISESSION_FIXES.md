# マルチセッション対応修正実施レポート

## 📋 修正概要

このドキュメントは、マルチモーダル画像検索アプリケーションでマルチセッション環境における問題を修正した内容をまとめたものです。

## 🚨 修正された問題

### 1. spaCyモデルの重複初期化問題（重大）

**問題**: 各セッションで個別にspaCy ja_ginzaモデルを初期化していたため、メモリ使用量の激増と初期化時間の長大化が発生

**修正内容**:
- `app/nlp_service.py` を新規作成し、シングルトンパターンでspaCyモデルを管理
- `app/search_query_generator.py` を修正し、共有NLPServiceを使用
- スレッドセーフな実装により、マルチセッション環境でも安全に動作

### 2. 一時ディレクトリの共有問題（中程度）

**問題**: 全セッションが同じ一時ディレクトリを使用し、ファイル名衝突の可能性があった

**修正内容**:
- セッション固有の一時ディレクトリ作成機能を実装
- `app/cleanup_service.py` を新規作成し、古いセッションディレクトリの自動クリーンアップ機能を追加
- 24時間以上経過したセッションディレクトリを定期的に削除

## 📁 修正されたファイル

### 新規作成ファイル

1. **`app/nlp_service.py`** - spaCyモデルのシングルトン管理
2. **`app/cleanup_service.py`** - 一時ディレクトリのクリーンアップ管理
3. **`MULTISESSION_FIXES.md`** - この修正レポート

### 修正されたファイル

1. **`app/search_query_generator.py`**
   - `self.nlp = spacy.load("ja_ginza")` → `self.nlp_service = NLPService()` に変更
   - 全ての `self.nlp(query)` → `nlp = self.nlp_service.get_nlp(); doc = nlp(query)` に変更

2. **`main.py`**
   - グローバルな一時ディレクトリ設定を削除
   - セッション固有の一時ディレクトリ作成機能を追加
   - クリーンアップサービスの統合

## 🔧 実装詳細

### NLPServiceクラス（シングルトンパターン）

```python
class NLPService:
    _instance = None
    _nlp = None
    _lock = threading.Lock()
    
    def get_nlp(self):
        if self._nlp is None:
            with self._lock:
                if self._nlp is None:
                    self._nlp = spacy.load("ja_ginza")
        return self._nlp
```

**特徴**:
- スレッドセーフなシングルトン実装
- 初回呼び出し時のみモデルをロード
- 全セッションで同じモデルインスタンスを共有

### セッション固有一時ディレクトリ

```python
def create_session_temp_dir():
    session_id = str(uuid.uuid4())[:8]
    temp_dir = os.path.join(os.getcwd(), "temp", f"session_{session_id}")
    os.makedirs(temp_dir, exist_ok=True)
    return temp_dir
```

**特徴**:
- UUID-8文字でセッション識別
- `temp/session_xxxxxxxx/` 形式のディレクトリ構造
- ファイル名衝突の完全回避

### 自動クリーンアップ機能

```python
cleanup_service = CleanupService(base_temp_dir, max_age_hours=24)
cleanup_service.start_cleanup_thread(interval_minutes=120)
```

**特徴**:
- 24時間以上経過したセッションディレクトリを自動削除
- 2時間間隔でクリーンアップ実行
- バックグラウンドスレッドで動作
